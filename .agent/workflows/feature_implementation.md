# 기능 구현 워크플로우 (Feature Implementation Workflow)

모든 중요한 코드 변경 시 이 프로세스를 따르십시오.

## 1. 계획 및 설계 (Planning Mode)

- [ ] **목표 이해**: `requirements.md` 및 사용자 요청 사항을 읽습니다.
- [ ] **맥락 파악**: 관련 파일(`README.md`, 기존 코드)을 읽습니다.
- [ ] **계획 수립**: `implementation_plan.md`를 작성합니다.
  - DB 스키마 변경 사항 정의.
  - API 엔드포인트 정의 (메서드, URL, 본문, 응답).
  - 검증 단계 정의.
- [ ] **사용자 검토**: `notify_user`를 사용하여 승인을 받습니다.

## 2. 구현 (Execution Mode)

- [ ] **DB 마이그레이션**: 스키마 변경을 먼저 적용합니다.
- [ ] **코드 변경**: 계획에 따라 구현합니다.
  - `docs/patterns/backend_conventions.md`를 따릅니다.
  - 함수는 작고 집중적으로 유지합니다.
- [ ] **명세 업데이트**: 해당되는 경우 Swagger/OpenAPI를 업데이트합니다.

## 3. 검증 (Verification Mode)

- [ ] **자동화 테스트**: 기존 테스트를 실행하거나 새 테스트를 생성합니다.
- [ ] **수동 검증**:
  - `curl` 또는 스크립트를 사용하여 API 엔드포인트를 테스트합니다.
  - DB 데이터 무결성을 확인합니다.
- [ ] **버그 수정**: 문제가 발견되면 수정하고 다시 검증합니다.

## 4. 문서화 및 정리

- [ ] **커밋 메시지**: 한글 커밋 메시지 제목을 제안합니다.
- [ ] **문서 업데이트**: 범위가 변경된 경우 `README.md` 또는 `requirements.md`를 업데이트합니다.
- [ ] **워크스루**: 결과와 함께 `walkthrough.md`를 생성/업데이트합니다.
