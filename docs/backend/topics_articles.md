# 토픽 및 기사 관리 (Topics & Articles Management)

## 1. 토픽 (`src/topics`)

토픽은 플랫폼의 중심 엔티티로, 뉴스 기사와 사용자 토론을 그룹화합니다.

### 데이터 모델 (`tn_topic`)

- `id`: 기본 키 (Primary Key).
- `display_name`: 토픽의 제목입니다.
- `status`: 가시성과 상호 작용을 제어합니다.
  - `PREPARING`: 생성되었지만 아직 사용자에게 보이지 않는 상태입니다.
  - `OPEN`: 공개되어 사용자가 투표하고 기사를 볼 수 있는 상태입니다.
  - `CLOSED`: 종료되었으며 선택적으로 보일 수 있습니다.
- `stance_left`, `stance_right`: 상반된 관점에 대한 설명입니다.

### 생명주기 (Lifecycle)

1.  **생성**: 관리자가 토픽을 생성합니다 (상태: `PREPARING`).
2.  **큐레이션**: 관리자가 기사를 수집하고 토픽에 할당합니다.
3.  **오픈**: 관리자가 상태를 `OPEN` 또는 `VOTING`으로 변경합니다.
4.  **상호 작용**: 사용자가 기사를 읽고, 투표하고, 채팅합니다.
5.  **종료**: 토픽이 종료되거나 발행됩니다.

## 2. 기사 (`src/articles`)

기사는 토픽에 연결된 개별 뉴스 항목입니다.

### 데이터 모델 (`tn_article`)

- `topic_id`: `tn_topic`에 대한 외래 키 (Foreign Key).
- `title`, `summary`, `url`: 기사 메타데이터입니다.
- `side`: 기사의 정치적 성향입니다 (`LEFT`, `RIGHT`, `CENTER`).
- `status`:
  - `suggested`: AI/스크립트에 의해 수집되었으나 아직 승인되지 않음.
  - `published`: 관리자에 의해 승인되어 사용자에게 보임.
  - `rejected`/`deleted`: 제거됨.

### 수집 및 할당

- **자동화**: `rss_collector.py`가 기사를 가져옵니다. `vector_indexer.py`는 의미론적 유사성을 기반으로 기사를 제안할 수 있습니다.
- **수동**: 관리자는 관리자 UI를 통해 수동으로 기사를 추가하거나 순서를 변경할 수 있습니다.

### 큐레이션 레이아웃 (Curation Layout)

토픽 상세 페이지는 3단 탭 인터페이스를 제공하여 다양한 시각의 기사를 큐레이션합니다:

- **진보 (Progressive)**: 좌측 성향의 기사
- **중도 (Center)**: 중립적 시각의 기사
- **보수 (Conservative)**: 우측 성향의 기사

관리자는 각 탭별로 기사 순서를 조정하여 사용자에게 균형 잡힌 시각을 제공할 수 있습니다.
