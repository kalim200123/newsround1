# 뉴스 커뮤니티 플랫폼 기능 명세서 (Feature Requirements)

## 🎨 UI/UX 개선 사항 (Global)

- **전체적인 UI 변경**: 기존 디자인을 전면적으로 개편하여 사용자 경험 개선.
- **채팅창 테두리 변경**: 실시간 채팅창의 시각적 경계 디자인 수정.
- **댓글 UI 개선**: '아하(Aha)' 서비스의 댓글 스타일을 벤치마킹하여 적용.

---

## 📄 페이지별 상세 명세

### 1. 메인 페이지 (Main Page)

- **레이아웃 배치**:
  - **좌측**: 실시간 채팅창 (ROUND1)
  - **중앙/왼쪽**: ROUND2 (주제 3~5개 노출)
  - **키워드 영역**: #키워드 3~5개 (두루뭉술하게 배치하거나 정렬)
- **변경 사항**:
  - 기존 '최신 기사' 섹션 삭제.
  - 하단 '카테고리별 기사' UI 변경 필요.

### 2. 카테고리 페이지 (Category Page)

- **기사 배치**: 언론사별로 그룹화하여 배치.
- **키워드**: 관련 키워드를 시각적으로 뿌려주는 UI 적용.

### 3. ROUND2 페이지 (토론 및 투표)

- **좌측 영역**:
  - **투표 기능**: 주제와 2개의 대립되는 주장 표시.
  - **통계**: 주장별 투표 비율(%) 비교 UI.
  - **댓글 창**: 투표 영역 하단에 활성화.
- **우측 영역 (기사 리스트)**:

  - **목적**: 토론을 위한 근거 자료(Reference) 제공에 집중.
  - **탭(Tab)**: '좌(진보) / 중(중도) / 우(보수)' 성향별 기사 리스트업.
  - **기사 카드 UI**:
    - **필수 정보**: 썸네일, 제목, 언론사, 날짜.
    - **액션**:
      - **기사 읽기**: 클릭 시 해당 기사로 이동.
      - **저장(Bookmark)**: 나중에 읽기 위한 스크랩 기능 (우측 상단 등 방해되지 않는 위치).
    - **제거**: 댓글 수, 좋아요/싫어요 버튼 (토론 집중을 위해 제거).

- **메인 댓글 기능 상세 (좌측 하단)**:
  - **표시 정보**: 프로필 이미지/URL, 닉네임, 작성 시각.
  - **주장 표시**: 작성자가 선택한 주장(입장)을 함께 표시.
  - **기능**: 코멘트 작성, 좋아요, 싫어요, 대댓글.
  - **인터랙션**: 댓글 클릭 시 바로 `@닉네임` 태그와 함께 답글 작성 모드 진입 (사진 첨부 기능 포함).

### 4. #키워드 페이지 (Keyword Page)

- **운영 방식**: 관리자가 직접 키워드 설정 (초기 단계).
- **동작**: 특정 키워드(예: `'#윤석열'`) 선택 시 해당 페이지로 이동.
- **레이아웃**:
  - **좌측**: 실시간 채팅창.
  - **우측**: 관련 기사 리스트업 (성향 구분 없이 전체 노출).

### 5. 검색 페이지 (Search Page)

- **진입**: 돋보기 아이콘 클릭 시 검색 페이지로 이동.
- **검색 결과**:
  - 검색어 입력 시 관련 **기사**, **ROUND2(토론)**, **ROUND3(채팅방)** 결과를 통합하여 보여줌.

### 6. ROUND3 페이지 (1:1 채팅)

- **범위**: 초기 단계에서는 1:1 채팅만 지원 (N:N 채팅은 복잡도 문제로 보류).
- **주요 기능**:
  - **친구 추가**: 사용자 간 친구 맺기.
  - **방 생성**: 주제, 주장(입장), 토론 시간 등을 설정하여 방 개설.
  - **관전자 모드**: 실시간 채팅이므로 Socket을 통해 제3자가 관전할 수 있는 기능 필요.

### 7. 알림 및 기타 (Notifications)

- **형태**: 별도 알림 페이지, 팝업창, 또는 모달창 형태로 구현.
